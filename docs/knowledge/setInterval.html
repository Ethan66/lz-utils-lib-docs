setInterval替代方案 ​

前言 ​

  • 为什么写这篇文章，因为需求时不时会让前端定时来请求后端接口来查询最新的数据。前端定时无非就2种，setTimeout和setInterval，一般都会选择setInterval，为什么，因为setInterval香啊。但是我还是推荐setTimeout，为什么，当然是因为setInterval有问题。

setInterval ​

setInterval什么问题 ​

  1. 某些时间间隔会被跳过。规定每隔1s请求接口一次，最终请求的时间是20::00:01, 20::00:03, 20::00:05，变成2s间隔请求一次
  2. 多个定时器会连续执行。还是规定每隔1s请求接口一次，最终请求的时间是20::00:01, 20::00:02, 20::00:02，有两个定时器执行时间一致

setInterval的实质 ​

setInterval指定的时间间隔，表示的是何时将定时器的代码添加到消息队列，而不是何时执行代码。

间隔会被跳过 ​

js
function task(num) {
  // 模拟主线任务执行过程耗时
  var startTime = Date.now();
  while (Date.now() - startTime < 2000) {}
}

// 定时器定时1s
var num = 0
var timer = setInterval(() => {
    num++
    if (num > 3) return clearInterval(timer)
    console.log(`开始第' ${num} '个任务：`, new Date().toLocaleTimeString());
    task(num)
}, 1000);

可以看到，虽然定时器执行间隔是1s，但是间隔变成了2s

多个定时器连续执行 ​

js
function longRunningTask(num) {
  // 模拟一个较长时间的任务
  var startTime = Date.now();
  while (Date.now() - startTime < 2000) {}
  console.log(`%c结束第' ${num} '个任务：`, 'background: red; color: #fff;', new Date().toLocaleTimeString());
}

// 定时器定时1s
var num = 0
var timer = setInterval(() => {
    num++
    if (num > 6) return clearInterval(timer)
    console.log(`%c开始第' ${num} '个任务：`, 'background: green; color: #fff;', new Date().toLocaleTimeString());
    longRunningTask(num)
}, 1000);

可以看到，定时器连续执行了，虽然间隔了1s，但是代码执行逻辑是无间隔的，不符合常理

优化1 -- setTimeout代替setInterval ​

setTimeout优点 ​

  1. 保证定时器间隔（解决问题一）
  2. 在前一个定时器执行完前，不会向队列插入新的定时器（解决问题二）
js
var timer = null
var a = performance.now()
function interval(func, wait){
    let interv = function(){
        const b = performance.now()
        console.log(b - a)
        a = b
         clearTimeout(timer)
        timer=setTimeout(interv, wait);
        func.call(null);
    };
    timer= setTimeout(interv, wait);
 }
var num = 0
interval(() => {num++; if (num > 4) clearTimeout(timer)}, 0)

这时出现，setTimeout的边界问题，简单来说，5 层以上的定时器嵌套会导致至少 4ms 的延迟。

优化2 -- setTimeout每次执行修正时间 ​

当每一次定时器执行时后，都去获取系统的时间来进行修正，虽然每次运行可能会有误差，但是通过系统时间对每次运行的修复，能够让后面每一次时间都得到一个补偿。

原时间差 ​

100ms重复100次，应该是10s，实际得到的是11s

修正时间后 ​

js
function interval(func, wait){
    var init = performance.now()
    let n = 0
    let interv = function(){
        n++
        const now = performance.now()
         clearTimeout(interval.timer)
        const diff = now - init - n * wait // 获取差值
        interval.timer=setTimeout(interv, wait - diff); // 设置实际wait
        func.call(null);
    };
    interval.timer= setTimeout(interv, wait);
 }
interval.timer = null

var num = 0
var initTime = new Date().getTime()
interval(() => {
    num++
    if (num > 100) {
        clearTimeout(interval.timer)
        console.log('最早时间戳: ', initTime)
        console.log('最新时间戳: ', new Date().getTime())
        console.log('差值: ', new Date().getTime() - initTime)
    }
}, 100)

原时间差 ​

100ms重复100次，应该是10s，实际得到的是10.1s

/**
 * setTimeout实现setInterval
 * @param callback 回调函数
 * @param wait 等待时间
 * @returns timerId
 */
export function interval(callback, wait): number {
  let timer = NaN
  var init = performance.now();
  let n = 0;
  let fn = function () {
    n++;
    const now = performance.now();
    clearTimeout(timer);
    const diff = now - init - n * wait; // 获取差值
    timer = window.setTimeout(fn, wait - diff); // 设置实际wait
    callback.call(null);
  };
  timer = window.setTimeout(fn, wait);
  return timer;
}

export default interval;
