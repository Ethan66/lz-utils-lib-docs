zod：运行时ts ​

使用场景 ​

  1. typescript只能进行编译时检查，运行时转为js，并没有校验类型的功能，所以需要使用运行时的类型校验。
  2. 测试环境：有些接口返回的字段太多，影响bug排查速度，或者字段值太过隐晦，比如约定返回'1','2'，结果返回了1, 2，所以知道了返回了非约定的字段值，直接把bug指派给后端。
  3. 线上环境：和后端约定了对应的字段值，防止后端在后续版本中返回了不符合约定的字段值，导致前端报错，让前端一度以为自己没有做好兜底，比如没有考虑字段值为null的情况。

基本类型 ​

ts
import { z } from 'zod';

const objSchema = z.object({
  name: z.string(),
  name2: z.enum(['a', 'b', 'c']), // 'a' | 'b' | 'c'
  age: z.number(),
  age2: z.union([z.literal(0), z.literal(1)]), // 1 | 2
  isStudent: z.boolean(),
  birthday: z.date(),
  null: z.null(),
  arr: z.array(z.string()),
}).passthrough()

一个属性值匹配多个类型 ​

ts
const objSchema = z.object({
  age: z.union([z.literal(0), z.literal(1), z.literal(false)]), // 0 | 1 | false
})

一个属性值匹配范围 ​

ts
const objSchema = z.object({
  age3: z.number().min(10).max(100), // 10-100
})

在已有的基础上选择部分属性 ​

ts
const fullSchema = z.object({
  name: z.string(),
  age: z.number().min(18),
  email: z.string().email(),
});

// 使用 .pick() 只验证 "name" 和 "age" 字段
const partialSchema = fullSchema.pick({
  name: true,
  age: true,
});

// 使用 .omit() 排除 "email" 字段，只验证 "name" 和 "age"
const partialSchema = fullSchema.omit({
  email: true,
});

api解释 ​

z.literal() ​

可以限制字段值必须等于value, 这个值可以是任意基本类型（字符串、数字、布尔值、null 等）

ts
const schema = z.literal("hello");
// 验证
schema.parse("hello"); // success

z.union() ​

用于创建一个联合类型验证规则。它非常适合需要验证值可以是多种不同类型的场景

ts
z.union([schema1, schema2, schema3, ...])
const schema = z.union([z.literal("small"), z.literal("large")]);

z.passthrough() ​

允许对象包含未在模式中定义的额外字段

ts
z.object({...}).passthrough();

z.optional() ​

可选字段

ts
const userSchema = z.object({
  name: z.string(),
  age: z.number().optional(),
});

实战 ​

接口响应数据校验公共方法 ​

ts
import { z } from 'zod'

export const validateData = <T>(schema: z.ZodType<T>, data: unknown): T | null => {
  try {
    return schema.parse(data);
  } catch (error) {
    if (error instanceof z.ZodError) {
      const errs = error.errors.reduce((errResult, cur) => {
        const val = cur.path.reduce((objVal: any, key) => objVal[key], data)
        const path = cur.path.join('.')
        errResult[path] = `val: ${val}, errMsg: ${cur.message}`
        return errResult
      }, {} as any)
      console.error('zod validate error: ', errs)
    } else {
      console.error('zod unexpected error:', error);
    }
    return data as any;
  }
};

使用 ​

ts
import { validateData } from './utils'

const originData = await fetch('https://api.example.com/data').json()
const userSchema = z.object({
  name: z.string(),
  age: z.number().optional(),
}).passthrough();

type userSchemaType = z.infer<typeof userSchema>

const data = validateData(userSchema, originData)
